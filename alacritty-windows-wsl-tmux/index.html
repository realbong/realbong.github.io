가벼운 터미널을 찾다가 Alacritty에 tmux를 붙였다

Windows 노트북에서 Alacritty를 WSL과 tmux 기반으로 쓰기 위한 설정 기록

WezTerm을 한동안 잘 쓰고 있었다. 기능도 많고 설정도 편했다. 문제는 성능이 좋지 않은 윈도우 노트북이었다. 가끔 CPU가 생각보다 높게 올라갔다. 터미널 하나 때문에 노트북이 버거워지면 계속 신경 쓰이게 된다. 그래서 다른 터미널을 찾다가 Alacritty를 알게 됐다.

Alacritty를 봤을 때 든 생각

Alacritty는 단순하다. 그리고 빠르다. 이 두 가지는 마음에 들었다. 다만 너무 단순했다. 탭이 없고, 화면 분할도 자체 기능으로 제공하지 않는다. 가벼운 건 좋은데 매일 쓰기에는 불편하지 않을까 싶었다.

그러다 John Grib 님의 Alacritty 글을 봤다.

핵심은 간단했다.

  • Alacritty는 터미널 에뮬레이터 역할만 가볍게 한다.
  • 탭과 화면 분할은 tmux가 담당한다.
  • Alacritty 단축키로 tmux의 탭과 화면 분할을 조작한다.

좋은 방향이라고 느꼈다. 다만 글의 설정은 macOS 기준이었다. 나는 Windows 노트북에서 WSL을 쓰고 있으니, Windows + WSL 기준으로 다시 맞춰야 했다.

목표

내가 원한 건 이 정도였다.

  • Alacritty를 실행하면 PowerShell이 아니라 WSL로 바로 들어간다.
  • WSL에 들어가면 tmux 세션에 자동으로 붙는다.
  • tmux window를 터미널 탭처럼 쓴다.
  • tmux pane을 화면 분할처럼 쓴다.
  • 단축키는 Windows에서 쓰기 편하게 Ctrl + Shift 조합으로 맞춘다.

핵심은 Alacritty 자체를 복잡하게 만들지 않는 것이다. 부족한 기능은 tmux에 맡긴다.

Alacritty에서 WSL과 tmux 실행하기

설정 파일 위치

Windows에서 Alacritty 설정 파일은 보통 여기에 둔다.

C:\Users\<사용자>\AppData\Roaming\alacritty\alacritty.toml

내 환경에서는 다음 경로였다.

C:\Users\MZ02- BONGIK\AppData\Roaming\alacritty\alacritty.toml

tmux 세션에 바로 붙기

먼저 Alacritty가 실행될 때 WSL로 들어가고, 바로 tmux main 세션에 붙게 했다.

[terminal]
shell = { program = "wsl.exe", args = ["--cd", "~", "-e", "tmux", "new-session", "-A", "-s", "main"] }

중요한 부분은 tmux new-session -A -s main이다. -s main은 세션 이름을 main으로 두는 옵션이고, -A는 이미 main 세션이 있으면 새로 만들지 않고 바로 붙는다.

즉 Alacritty를 여러 번 열어도 같은 tmux 세션을 이어서 쓸 수 있다.

탭처럼 쓰기

Alacritty에는 탭이 없다. 대신 tmux window를 탭처럼 쓰면 된다.

Alacritty 단축키

자주 쓰는 기능은 Alacritty 단축키로 바로 동작하게 했다.

[keyboard]
bindings = [
  { key = "T", mods = "Control|Shift", chars = "\u0011c" },
  { key = "W", mods = "Control|Shift", chars = "\u0011X" },
  { key = "!", mods = "Control|Shift", chars = "\u00111" },
  { key = "@", mods = "Control|Shift", chars = "\u00112" },
  { key = "#", mods = "Control|Shift", chars = "\u00113" },
  { key = "$", mods = "Control|Shift", chars = "\u00114" },
  { key = "%", mods = "Control|Shift", chars = "\u00115" },
  { key = "^", mods = "Control|Shift", chars = "\u00116" },
  { key = "&", mods = "Control|Shift", chars = "\u00117" },
  { key = "*", mods = "Control|Shift", chars = "\u00118" },
  { key = "(", mods = "Control|Shift", chars = "\u00119" },
  { key = "E", mods = "Control|Shift", chars = "\u0011w" },
]

단축키는 이렇게 잡았다.

  • Ctrl + Shift + t: 새 탭 생성
  • Ctrl + Shift + w: 현재 pane 닫기
  • Ctrl + Shift + 19: 19번 탭으로 이동
  • Ctrl + Shift + e: 탭 목록 보기

Windows 키보드에서는 Shift + 1이 !가 되는 식으로 숫자와 Shift 조합이 기호 문자로 들어간다. Alacritty 키 바인딩도 이 문자 기준으로 맞춰야 해서 설정에는 1 대신 !, @, # 같은 문자를 적었다.

tmux window 설정

tmux 쪽에는 window 번호를 1부터 시작하도록 설정했다.

set -g base-index 1
setw -g pane-base-index 1
set-option -g renumber-windows on
bind c new-window -c "#{pane_current_path}"

renumber-windows on을 켜두면 중간 탭을 닫아도 번호가 다시 정리된다. 탭처럼 쓰려면 이 편이 훨씬 자연스럽다. 새 탭도 현재 작업 디렉터리에서 열리도록 했다. 프로젝트 폴더에서 새 탭을 열었는데 홈 디렉터리로 돌아가 있으면 흐름이 자주 끊기기 때문이다.

화면 분할처럼 쓰기

화면 분할도 tmux pane으로 처리했다.

분할 단축키

{ key = "ArrowLeft", mods = "Control|Shift", chars = "\u0011%" },
{ key = "ArrowRight", mods = "Control|Shift", chars = "\u0011%" },
{ key = "ArrowUp", mods = "Control|Shift", chars = "\u0011\"" },
{ key = "ArrowDown", mods = "Control|Shift", chars = "\u0011\"" },

단축키는 이렇게 쓴다.

  • Ctrl + Shift + ←: 좌우 분할
  • Ctrl + Shift + →: 좌우 분할
  • Ctrl + Shift + ↑: 상하 분할
  • Ctrl + Shift + ↓: 상하 분할

좌우와 상하를 d, s 같은 문자로 외우기보다 방향키로 묶는 편이 더 직관적이었다. tmux 설정은 그대로 두고, Alacritty가 tmux prefix와 분할 키를 보내는 방식만 바꿨다.

prefix key

pane 이동은 Alacritty 단축키로 우회하지 않고 tmux 기본 방식으로 쓴다.

  • Ctrl + q -> o: 다음 pane으로 이동
  • Ctrl + q -> Ctrl + o: pane 순서 전환

새로운 pane 생성

tmux에서는 새 pane이 현재 작업 디렉터리에서 열리도록 했다.

bind % split-window -h -c "#{pane_current_path}"
bind '"' split-window -v -c "#{pane_current_path}"
bind-key X confirm-before -p "kill pane? (y/n)" kill-pane

이 설정을 하지 않으면 새 pane이 홈 디렉터리에서 열려서 흐름이 자주 끊긴다. 터미널을 자주 나누는 사람에게는 꽤 중요한 부분이다. pane을 닫을 때는 실수로 날리지 않도록 한 번 확인하게 했다.

pane 크기 조절

마우스로 조절

pane 크기 조절은 단축키를 따로 두지 않고 마우스로 처리한다. tmux에서 마우스를 켜두면 pane 경계선을 직접 잡아당길 수 있다.

set -g mouse on

분할은 키보드로 빠르게 만들고, 크기 조절은 눈으로 보면서 마우스로 맞추는 편이 지금 방식에는 더 편하다.

tmux 상태바 위치

상태바를 위로 올리기

tmux 아래쪽에 있던 상태바는 위로 올렸다. 보통 tmux status bar 또는 status line이라고 부른다.

set -g status-position top
set -g status-left "#{?client_prefix,#[reverse] PREFIX #[default],} #S "

상태바가 위에 있으면 터미널 탭을 보는 느낌에 조금 더 가깝다. 그리고 Ctrl + q를 눌렀을 때 왼쪽에 PREFIX가 표시되도록 했다. 지금 입력이 tmux 명령으로 들어갈 차례인지 눈으로 확인할 수 있어서 편하다.

정리

결국 한 일은 Alacritty에 없는 기능을 억지로 추가한 게 아니다.

Alacritty는 빠르고 단순한 터미널로 두고, 탭과 화면 분할은 tmux에 맡겼다. Windows + WSL 환경에서 쓰기 편하도록 실행 방식과 단축키만 연결했다.

지금 기준으로는 꽤 만족스럽다. WezTerm처럼 많은 기능을 한 번에 제공하는 도구도 좋지만, 성능이 애매한 노트북에서는 이렇게 역할을 나눠 쓰는 방식이 더 가볍게 느껴진다.
